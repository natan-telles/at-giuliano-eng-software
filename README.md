# 1 - Software Engineer at Google
> Quando falamos de programação, quase que instantaneamente, pensamos em escrever código, mas quando se trata de engenharia de software entramos em um âmbito mais complexo, já que esta engenharia se difere das outras convencionais. Engenharia de Software não se restringe a escrever códigos, mas engloba o uso de ferramentas, administração de processos, organização de ambientes, desde a construção até a manutenção de ambientes são fatores essenciais para a engenharia mais jovem das últimas décadas. Ademais, se discute constantemente acerca da programação integrada, que aponta para pilares como sustentabilidade contra mudanças ao longo de seu período de vida útil, escala e crescimento, e custos trade-offs . 


# 2 - Culture / Process / Tools

> Cultura (Cultura) -> Desenvolvimento de Times

> Tools (Ferramentas) -> Investimento em infraestrutura

# 3 - Non-function requirements
> Data Integrity: Proteger os dados, consistência, precisão.
> Security: Usar criptografia double-side para produção, prevenção contra atacantes mal-intencionados
> Scalability: Capacidade de lidar com alto volume de requisições simultaneamente
> Agility: Entrega incremental, colaboração contínua, adaptação à mudanças (entregar valor cedo, com alta qualidade)
> Deployability: Facilidade de deploy e colocar em produção


# 4 - Describe 3 scenarios of Trade-Offs (Software)
> Consistency X Availabilitdy : Erros em transações bancárias
> Speed X Quality: Startup/Escalável
> Performance X Simplicity: HashTable/loops & if's
